接口概览

GET/turnstile

提交求解任务，返回 `taskId`。

GET/result

根据 `taskId` 轮询结果，直到获得 `solution.token`。

提交求解任务

GET/turnstile?url=https://example.com&sitekey=0x4AAAA

`/turnstile` 与 `/result` 现在都必须携带 API Key；支持 Authorization: Bearer ts_xxx、X-API-Key: ts_xxx 或查询参数 api_key。

参数说明必填
url目标页面地址是
sitekeyTurnstile 站点 Key是
action可选动作参数否
cdata可选自定义数据否
 
curl "http://127.0.0.1:5072/turnstile?url=https://example.com/login&sitekey=0x4AAAAAACgP975UcSubdv3v"
import requests

res = requests.get(
    "http://127.0.0.1:5072/turnstile",
    params={
        "url": "https://example.com/login",
        "sitekey": "0x4AAAAAACgP975UcSubdv3v"
    }
).json()

print(res)
const params = new URLSearchParams({
  url: 'https://example.com/login',
  sitekey: '0x4AAAAAACgP975UcSubdv3v'
});

const res = await fetch(`http://127.0.0.1:5072/turnstile?${params}`);
console.log(await res.json());
<?php
$query = http_build_query([
    'url' => 'https://example.com/login',
    'sitekey' => '0x4AAAAAACgP975UcSubdv3v',
]);

$result = file_get_contents("http://127.0.0.1:5072/turnstile?$query");
echo $result;

查询任务结果

GET/result?id=<taskId>

curl "http://127.0.0.1:5072/result?id=task-id-here"
import requests, time

task_id = 'task-id-here'
while True:
    data = requests.get('http://127.0.0.1:5072/result', params={'id': task_id}).json()
    print(data)
    if data.get('status') != 'processing':
        break
    time.sleep(1)
const taskId = 'task-id-here';
const timer = setInterval(async () => {
  const res = await fetch(`http://127.0.0.1:5072/result?id=${taskId}`);
  const data = await res.json();
  console.log(data);
  if (data.status !== 'processing') clearInterval(timer);
}, 1000);
<?php
$taskId = 'task-id-here';
$result = file_get_contents("http://127.0.0.1:5072/result?id={$taskId}");
echo $result;

返回格式

创建任务成功

{
  "taskId": "f0dbe75b-fa76-41ad-89aa-4d3a392040af"
}

处理中

{
  "status": "processing"
}

成功结果

{
  "status": "ready",
  "solution": {
    "token": "0.xxx"
  },
  "elapsed_time": 7.62
}

失败结果

{
  "status": "failed",
  "value": "CAPTCHA_FAIL",
  "elapsed_time": 30.0
}

接入建议

  • 提交任务后使用 1 秒轮询间隔，避免无意义高频请求。
  • 生产环境建议把服务地址、重试策略和超时配置放入统一配置中心。
  • 如果你有多业务线接入，建议通过 API Key、IP 白名单和调用日志做隔离治理。
  • 若需要内部运营能力，请单独进入管理员后台，不在公共文档暴露后台接口。
